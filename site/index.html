Sync

Sync

A small networking helper that pairs a RemoteEvent and a RemoteFunction per endpoint name, with rate limiting + payload byte limiting.

Important: keep your endpoint definitions in one shared “Remotes” module required by both server and client. All endpoints share the same folder: ReplicatedStorage.Remotes.

What it does

  • One endpoint name = one RemoteEvent + one RemoteFunction
  • Rate limiting per endpoint (Throttle or Burst)
  • Byte limiting (fast estimate; oversized payloads are dropped)

Minimal example

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Sync = require(ReplicatedStorage.Packages.Sync)

local Ping = Sync:Create("Test.Ping")
  :SetRateLimit(2)
  :SetByteLimit(64)
  :SetMode(Sync.Modes.Burst)

Server

Ping:OnEvent(function(player, message)
  print("Ping from", player.Name, message)
end)

Client

Ping:FireServer("hello")

Recommended pattern: a Remotes module

Instead of creating Sync endpoints scattered across your codebase, make a single ModuleScript that returns a table of endpoints. This keeps names consistent on both server and client. 

-- ReplicatedStorage/Shared/Remotes.luau
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Sync = require(ReplicatedStorage.Packages.Sync)

return {
  Test = {
    Ping = Sync:Create("Test.Ping"):SetRateLimit(2):SetByteLimit(64),
  },

  Movement = {
    -- High-frequency state: use FireUnreliable (drops are OK).
    Position = Sync:Create("Movement.Position"):SetRateLimit(30):SetByteLimit(512),
  },
}

Server usage

local Remotes = require(game:GetService("ReplicatedStorage").Shared.Remotes)

Remotes.Test.Ping:OnEvent(function(player, message)
  print("Ping from", player.Name, message)
end)

Client usage

local Remotes = require(game:GetService("ReplicatedStorage").Shared.Remotes)

Remotes.Test.Ping:FireServer("hello")

Unreliable events

Each Sync endpoint includes both a reliable RemoteEvent and an UnreliableRemoteEvent. Use :FireServer(...) for reliable delivery, and :FireUnreliable(...) for high-frequency updates where it’s okay to drop packets.

Client → Server (unreliable)

local Remotes = require(game:GetService("ReplicatedStorage").Shared.Remotes)

-- Send position updates; some packets may drop (expected).
Remotes.Movement.Position:FireUnreliable(workspace.CurrentCamera.CFrame)

Server listen (unreliable)

Remotes.Movement.Position:OnEvent(function(player, cf)
  -- cf is a CFrame (or whatever you send)
end)
