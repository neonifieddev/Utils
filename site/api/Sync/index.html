API

Sync

Sync

Basic networking module for creating and handling static RemoteEvents and RemoteFunctions (paired per endpoint name), with rate limiting and payload byte limiting. Each endpoint also includes an UnreliableRemoteEvent for high-frequency state.

Examples only: this page uses generic examples (no app-specific remote registries), inspired by the structure of RbxUtil’s Net docs.

One shared remote folder

You only need one copy of the Sync module required by both server and client. All endpoints share: ReplicatedStorage.Remotes.

For each Sync:Create("X"):

  • RemoteEvent: ReplicatedStorage.Remotes.X
  • UnreliableRemoteEvent: ReplicatedStorage.Remotes.XUnreliable
  • RemoteFunction: ReplicatedStorage.Remotes.XFunction

Quick example

local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Sync = require(ReplicatedStorage.Packages.Sync)

local PointsChanged = Sync:Create("PointsChanged")
  :SetRateLimit(10)
  :SetByteLimit(1024)
  :SetMode(Sync.Modes.Burst)

Client

PointsChanged:OnEvent(function(points)
  print("Points", points)
end)

Server

PointsChanged:FireAllClients(10)

Recommended pattern: return a table of endpoints

A common pattern is a shared ModuleScript (required by both server + client) that exports a table of Sync endpoints. 

-- ReplicatedStorage/Shared/Remotes.luau
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local Sync = require(ReplicatedStorage.Packages.Sync)

return {
  Test = {
    Ping = Sync:Create("Test.Ping"):SetRateLimit(2):SetByteLimit(64),
  },

  Data = {
    Get = Sync:Create("Data.Get"):SetRateLimit(10):SetByteLimit(2048):SetMode(Sync.Modes.Burst),
  },

  Movement = {
    -- Same endpoint can be fired reliably or unreliably.
    Position = Sync:Create("Movement.Position"):SetRateLimit(30):SetByteLimit(512),
  },
}

Server usage

local Remotes = require(ReplicatedStorage.Shared.Remotes)

Remotes.Test.Ping:OnEvent(function(player, msg)
  print(player.Name, msg)
end)

Remotes.Data.Get:OnInvoke(function(player, key)
  return 123
end)

Client usage

local Remotes = require(ReplicatedStorage.Shared.Remotes)

Remotes.Test.Ping:FireServer("hello")
local n = Remotes.Data.Get:InvokeServer("Cash")

Functions

Create

Sync:Create(name: string) -> Sync

Creates (or returns) a Sync endpoint.

  • Server: creates the endpoint remotes under ReplicatedStorage.Remotes: RemoteEvent, UnreliableRemoteEvent, and RemoteFunction.
  • Client: waits up to 10 seconds for the folder/remotes to exist.
  • Duplicate name: warns and returns the existing instance.
local sync = Sync:Create("Test.Ping")

Init

Sync:Init() -> ()

One-time module initializer. Sync:Create() calls this automatically.

InitServer

Sync:InitServer() -> () Server

InitClient

Sync:InitClient() -> () Client

Modes

Sync.Modes = {
  Throttle = "Throttle",
  Burst = "Burst",
}
  • Throttle: minimum interval between calls (1 / rateLimit seconds).
  • Burst: up to rateLimit calls per second.

Instance Methods

OnEvent

sync:OnEvent(callback: (...any) -> ()) -> { Disconnect: () -> () }?

Connects a handler to the endpoint’s events. For simplicity, :OnEvent() receives both reliable and unreliable event traffic. 

-- Client
sync:OnEvent(function(...)
  print("Got", ...)
end)

-- Server
sync:OnEvent(function(player, ...)
  print("From", player.Name, ...)
end)

OnInvoke

sync:OnInvoke(callback: (...any) -> ...any) -> { Disconnect: () -> () }?

-- Server
sync:OnInvoke(function(player, request)
  return { ok = true }
end)

FireServer

sync:FireServer(...any) -> () Client

Fires client → server (drops if rate-limited or byte-limited).

sync:FireServer("hello")

FireUnreliable

sync:FireUnreliable(...any) -> () Client

Fires client → server using UnreliableRemoteEvent (drops are expected; still rate/byte limited).

sync:FireUnreliable(workspace.CurrentCamera.CFrame)

InvokeServer

sync:InvokeServer(...any) -> ...any Client

local result = sync:InvokeServer("GetData")

FireAllClients

sync:FireAllClients(...any) -> () Server

sync:FireAllClients("announcement")

FireAllUnreliable

sync:FireAllUnreliable(...any) -> () Server

Fires to all players using UnreliableRemoteEvent (per-player rate limiting; drops expected).

sync:FireAllUnreliable(CFrame.new())

FireClient

sync:FireClient(player: Player, ...any) -> () Server

sync:FireClient(player, "hello")

FireClientUnreliable

sync:FireClientUnreliable(player: Player, ...any) -> () Server

sync:FireClientUnreliable(player, CFrame.new())

FireAllClientsExcept

sync:FireAllClientsExcept(players: { Player }, ...any) -> () Server

sync:FireAllClientsExcept({playerA, playerB}, "everyone else")

FireExceptUnreliable

sync:FireExceptUnreliable(players: { Player }, ...any) -> () Server

sync:FireExceptUnreliable({playerA, playerB}, CFrame.new())

InvokeClient

sync:InvokeClient(player: Player, ...any) -> ...any Server

local ok = sync:InvokeClient(player, "Ping")

Destroy

sync:Destroy() -> ()

Destroys server-created remotes (server) and cleans connections/tasks (all realms).

Configuration

SetRateLimit

sync:SetRateLimit(perSecondLimit: number) -> Sync

sync:SetRateLimit(15)

SetMode

sync:SetMode(mode: string) -> Sync

sync:SetMode(Sync.Modes.Burst)

SetByteLimit

sync:SetByteLimit(byteLimit: number) -> Sync

sync:SetByteLimit(1024)

Unreliable vs reliable

  • Reliable (RemoteEvent): use for critical gameplay messages.
  • Unreliable (UnreliableRemoteEvent): use for high-frequency state where drops are OK.
